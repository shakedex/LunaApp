## Third-Party SDKs

Luna integrates with several proprietary, vendor-supplied SDKs and tools to enable native processing for specific camera families. **Luna does not bundle or redistribute these SDKs.** The end user is responsible for obtaining and installing them under each vendor's respective license.

| Vendor             | Component                  | Distribution                                   |
| ------------------ | -------------------------- | ---------------------------------------------- |
| ARRI               | ARRI ART CLI (`art-cmd`)   | User downloads from ARRI; placed in `tools/arri/<rid>/` |
| Blackmagic Design  | Blackmagic RAW SDK         | User installs from Blackmagic Developer site   |
| Sony               | Sony RAW Viewer            | User installs from Sony Pro support site       |

If any of these are not present on the user's machine, Luna's corresponding `ICameraSupport` will report `SupportStatus.NotAvailable` and emit an `UnsupportedFormatNotice` for affected clips. Generic FFmpeg-based decoding remains available as a fallback for many container formats.

## Code Signing

Luna binaries are **not currently code-signed**. As a result:

- **Windows** will display a SmartScreen warning on first run. See [docs/INSTALLATION.md](docs/INSTALLATION.md) for the standard "More info → Run anyway" flow.
- **macOS** will display a Gatekeeper warning. The installation guide covers the right-click → Open workaround.

This is normal for indie / unsigned software. If you would prefer to verify the code yourself, build from source — see the README.
